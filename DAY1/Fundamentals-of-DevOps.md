Usually these questions are asked when there is a DevOps engineer Interview

What is DevOps?  — DevOps is a culture and way of working that Organizations adopt to deliver products and applications with high quality and reduced time to market. Improves the Ability of an Organization to deliver the products and applications.

— Whenever there is a bug in software, the fixing should take least amount of time so that the business doesn’t lose revenue due to these scenarios. So less time to solve scenarios and time to delivery of the product to the market.

What is DevOps all about? 
Improving Delivery	Continuous Integration and Continuous Delivery
Automation	To Improve Delivery, Including Automation is also required as a Company delivering Products
Quality	Apart from Automation, The customer should also be provided with the Quality. 
Monitoring	Also About Monitoring, Ensuring the quality needs Monitoring. Issues needs to be reported which gets covered due to monitoring. 
Testing	So, Testing is also important, because without it we can’t ensure the quality and the automation.

— So, Now DevOps is a process of Improving delivery, by ensuring proper automation, proper quality, with continuous monitoring and testing.

— DevOps is a Process of Improving the application delivery by ensuring that there is proper Automation in place, by ensuing that there is a Quality which has been maintained, with Continuous Monitoring/Observability and Continuous Testing.   — The end goal as a DevOps engineer is that, there should not be any Manual Process and also fasten up the process of application delivery.


Why DevOps?

— Assuming there is a Developer — he is writing code for his application — end goal for this developer will be that it goes to the customer, who uses this application.

— Now what happens in between — before 10 years — when DevOps was not there, there will be multiple developers, so they put the code in a central location, the build and release engineer, at that time to create servers to deploy the app on that server, and then there will be a tester who will test this application on that server which was created by the BRE, after testing, the application will be promoted to staging or pre-prod environment, again after testing it will taken to the prod server.   End goal — (Dev —> Customer)  Because multiple parties were involved, for simple delivering of the application, it was involving too many parties which was delaying the time to market of the application.  So to Automate this entire process DevOps came into picture.
